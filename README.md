# 船厂焊接工位开工许可系统

## 项目简介

本系统将**焊工资格管理**、**工艺卡管理**和**安全检查**集成到同一套应用中，实现船厂焊接工位的全流程开工许可管控。系统覆盖班组长排班、焊工确认工艺卡、质检员首件检查等核心业务场景，并提供完整的状态流转和业务校验机制。

## 技术栈

### 后端
- **框架**: Spring Boot 2.7.18
- **ORM**: MyBatis-Plus 3.5.5
- **数据库**: MySQL 8.x
- **Java**: JDK 1.8+
- **其他**: Lombok, FastJSON

### 前端
- **框架**: Vue 3.4
- **构建工具**: Vite 5
- **UI 组件库**: Element Plus 2.5
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **HTTP 客户端**: Axios

## 核心功能

### 1. 焊工资格管理
- 焊工档案维护（编号、姓名、身份证、入职日期等）
- 资格证书管理（证书类型、等级、材质范围、有效期等）
- 证书状态自动刷新（有效、即将过期、已过期）
- 焊工资格有效性实时校验

### 2. 工艺卡管理
- 工艺卡全生命周期：草稿 → 已审批 → 已下发 → 已作废
- 焊接参数维护（电流、电压、速度、温度、气体等）
- 工艺卡下发后焊工才能查看和确认
- 工艺卡未下发不能排班开工

### 3. 工位管理
- 工位档案维护（工位编号、区域、设备信息）
- 工位实时状态：空闲 → 已排班 → 开工中 → 首检中 → 已暂停

### 4. 排班与开工许可（核心）
- **班组长排班**: 指定工位、焊工、工艺卡、班次、时间
- **开工前三重校验**:
  - ✅ 焊工资格是否在有效期内（过期不能开工）
  - ✅ 工艺卡是否已下发且焊工已确认（未确认不能开工）
  - ✅ 安全检查是否全部合格
- **焊工工艺卡确认**: 焊工查看工艺卡参数后确认/拒绝
- **安全检查**: 设备、防护、环境、消防等检查项

### 5. 首件质检
- 质检员记录外观检查、尺寸检查、无损检测结果
- 综合判定合格/不合格
- **首件不合格自动暂停后续工序**
- 支持复检流程

### 6. 工序暂停管理
- 暂停原因记录（首件不合格、安全问题、资格过期、其他）
- 暂停状态跟踪
- 解除暂停需填写解除措施

## 快速开始

### 1. 初始化数据库

```bash
# 创建数据库并导入初始数据
mysql -u root -p < database/schema.sql
```

### 2. 修改后端配置

编辑 `backend/src/main/resources/application.yml`，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/shipyard_welding?...
    username: root
    password: your_password
```

### 3. 启动后端

```bash
cd backend
mvn spring-boot:run
```

后端服务将在 `http://localhost:8080/api` 启动。

### 4. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端服务将在 `http://localhost:3000` 启动。

## 业务流程说明

```
班组长排班
    ↓
工位状态：空闲 → 已排班
    ↓
焊工确认工艺卡 + 安全检查
    ↓
开工许可校验（资格/工艺卡/安全检查）
    ↓ 校验通过
焊工开工（工位：已排班 → 开工中）
    ↓
提交首件检查（工位：开工中 → 首检中）
    ↓
质检员首件检查
    ├─ 合格 → 恢复开工（工位：首检中 → 开工中）
    └─ 不合格 → 自动暂停（工位：首检中 → 已暂停）+ 记录暂停单
    ↓
完工（工位：开工中 → 空闲）
```

## 关键业务规则

1. **焊工资格过期 → 不能开工**：开工前校验焊工是否有有效证书
2. **工艺卡未下发 → 不能排班**：排班时自动校验工艺卡状态
3. **工艺卡未确认 → 不能开工**：焊工必须先确认工艺卡参数
4. **首件不合格 → 暂停后续工序**：质检不合格自动触发暂停机制
5. **暂停解除后才能恢复作业**：必须填写解除措施
